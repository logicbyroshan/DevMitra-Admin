üìö Portfolio API Documentation

Read-only REST API for accessing portfolio data

üîí Security & Authentication

‚ö†Ô∏è Important Security Notes

  • Read-Only: All API endpoints are READ-ONLY. No write operations (POST, PUT, DELETE) are allowed from external sources.
  • CORS: API is restricted to allowed origins only. Update CORS_ALLOWED_ORIGINS in settings.py.
  • Rate Limiting: Anonymous users: 100 requests/hour
  • Data Filter: Only active and published content is exposed via API

Base URL

http://localhost:8000/api/

üìÅ Projects

Get all active projects
category Filter by category slug (optional)
status Filter by status: draft, in_progress, published (optional)
GET /api/projects/{slug}/

üíª Usage Example (JavaScript/Fetch)

// Fetch all projects
fetch('http://localhost:8000/api/projects/')
  .then(response => response.json())
  .then(data => {
    console.log('Projects:', data.results);
  })
  .catch(error => console.error('Error:', error));

// Fetch featured projects
fetch('http://localhost:8000/api/projects/featured/')
  .then(response => response.json())
  .then(data => {
    console.log('Featured projects:', data);
  });

// Fetch portfolio summary
fetch('http://localhost:8000/api/summary/')
  .then(response => response.json())
  .then(data => {
    console.log('Stats:', data);
  });

‚öôÔ∏è Configuration for Production

1. Update CORS Origins

In config/settings.py, add your portfolio website URL:

CORS_ALLOWED_ORIGINS = [
    "https://yourportfolio.com",
    "https://www.yourportfolio.com",
]

2. Generate Secure API Key (Optional)

Generate a random key and update in settings.py:

API_KEY = 'your-super-secure-random-key-here'

3. Enable HTTPS in Production

Ensure your portfolio site uses HTTPS for secure communication.
